  <action>
Create `components/products/VariantSelector.tsx` as a `'use client'` component.

Props interface:
```typescript
interface VariantSelectorProps {
  currentSlug: string;      // e.g. "mango-ice-4mg"
  currentFlavor: string;    // e.g. "Mango Ice"
  currentStrength: string;  // e.g. "4mg"
  region: string;           // for navigation prefix
}
```

Implementation:

1. **Fetch sibling variants on mount** using two parallel fetch calls inside a `useEffect`:
   - `fetch('/api/products?flavor=' + encodeURIComponent(currentFlavor))` — all strengths for this flavour
   - `fetch('/api/products?strength=' + encodeURIComponent(currentStrength))` — all flavours at this strength

   Store results in two state arrays: `strengthVariants: Product[]` (same flavour, different strengths) and `flavourVariants: Product[]` (same strength, different flavours).

2. **Strength button group** — rendered when `strengthVariants.length > 1`:
   - Label: "Strength" above the group
   - One `<button>` per variant, text = `product.nicotineStrength` (e.g. "4mg")
   - Active button (where slug === currentSlug): solid dark background (`bg-gray-900 text-white`)
   - Inactive: outlined (`border border-gray-300 text-gray-700 hover:border-gray-900`)
   - onClick: `router.push('/' + region + '/products/' + product.slug)` using `useRouter` from `next/navigation`
   - Sort strength variants ascending by numeric mg value before rendering

3. **Flavour button group** — rendered when `flavourVariants.length > 1`:
   - Label: "Flavour" above the group
   - One `<button>` per variant, text = `product.flavor`
   - Same active/inactive styling as strength group
   - Active: where slug === currentSlug
   - onClick: navigate to that variant's slug (same region prefix)

4. **Loading state**: while fetching, render a small skeleton (two rows of 4 grey rounded rectangles, `animate-pulse`).

5. **No results / single variant**: if only 1 strength variant and 1 flavour variant, render nothing (return null) — this product has no siblings.

Do NOT use `useRegion()` in this component — `region` is passed as a prop from the server component PDP page so it works correctly without a context boundary issue. Import only `useState`, `useEffect` from react and `useRouter` from `next/navigation`.
